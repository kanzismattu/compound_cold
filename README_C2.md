# Network Rail Weather Incident Analysis

Code supporting the analysis of weather-related infrastructure incidents on the
Scottish rail network, including their association with compound cold events and
large-scale atmospheric weather patterns.

---

## Data

### Network Rail incident data
Weather-related incident records from Network Rail. The dataset is not publicly
distributed. Please refer to the data availability statement in the paper for
access instructions.

The processed file `all_data.pkl` is a Pandas DataFrame with (at minimum) the
following columns:

| Column | Description |
|--------|-------------|
| `Date` | Incident date |
| `Incident Reason Name` | Weather incident category |
| `WP Number` | Met Office weather-pattern classification (1–30) |
| `PfPI Minutes` | Performance minutes lost (impact metric) |
| `Incident Duration` | Duration of the incident |
| `Lat`, `Long` | Incident location coordinates |

### Meteorological data
- **HadUK-Grid** (5 km) — daily minimum temperature and precipitation, available
  from the CEDA archive:  
  https://catalogue.ceda.ac.uk/uuid/4dc8450d889a491ebb20e724debe2dfb
- **Weather-pattern classification** — 30-pattern daily catalogue from the
  Met Office (see paper for reference).
- Monthly pre-processed arrays (`{month}_min_temp_all.npy`, `{month}_precip_all.npy`)
  and percentile threshold files are generated by the extraction notebooks in
  the companion repository (Paper 1).

### Rail network shapefile
`scotland_rail.shp` contains the Scottish rail network geometry in WGS84
(EPSG:4326).

## Running the code

### 1. Configure paths

Each notebook has a clearly marked **"User configuration"** section at the top.
Set `DATA_DIR`, `ARRAYS_DIR`, `PERCENTILES_DIR`, and `HADUK_SAMPLE_FILE` to
point to your local data before running.

### 2. Run order

| Step | Notebook | Description |
|------|----------|-------------|
| 1 | `01_dataset_analysis.ipynb` | Load data, produce summary figures, identify and save high-impact days |
| 2 | `02_sequence_footprints_plotting.ipynb` | Load high-impact day metadata, compute compound-event footprints, plot 5-day sequences |

### 3. Weather-pattern codes in notebook 2

For days in the 5-day window that are not themselves high-impact days, the
weather-pattern number is not automatically extracted from the dataset. Section
2a of `02_sequence_footprints_plotting.ipynb` contains a cell where WP values
can be entered manually for each event window.

## Notes

- The 99th-percentile threshold is used to define "high-impact" days in both
  incident-count and PfPI-minutes analyses. This is set via `PERCENTILE_THRESHOLD`
  in `01_dataset_analysis.ipynb` and can be adjusted.
- Weather-pattern groups follow a 30-pattern classification; the broad
  groupings (Anticyclonic, Cyclonic, Unbiased) and the finer regime ordering
  (NAO−, NAO+, NW, SW, SH, HP, LC, A) used in the plots are defined as
  constants in notebook 1.
