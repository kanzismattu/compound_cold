# High-Risk Pattern Identification and Compound-Event Forecasting

Code supporting the identification of high-risk weather patterns (HRPs) for
compound cold events on the Scottish rail network, and the verification of
ECMWF ensemble compound-event forecasts against observed high-impact days.

## Notebook descriptions and run order

| Step | Notebook | Source notebooks merged | Description |
|------|----------|------------------------|-------------|
| 1 | `01_WP_footprints_HRP_analysis.ipynb` | `WP_footprints_incident_days.ipynb` + `HRP_analysis.ipynb` | Computes per-day CD/CW footprint arrays for each WP subset, stacks them, then plots the modal (dominant) classification for all WPs and for the high-risk subset |
| 2 | `02_WP_forecast_probability_plots.ipynb` | `WP_probabilities_plot_v2__1_.ipynb` | Reads ECMWF WP ensemble CSV files and plots HRP forecast probabilities vs. climatology at 10-, 5- and 3-day lead times |
| 3 | `03_ecmwf_ensemble_processing.ipynb` | `ecmwf_probabilities__1_.ipynb` | Processes raw ECMWF GRIB ensemble files for each member, applies HadUK-Grid percentile thresholds, and saves binary CD/CW forecast cubes as NetCDF |

Notebooks 2 and 3–4 can be run in parallel (they are independent).

## Data

### HadUK-Grid observational data
Monthly temperature and precipitation arrays and percentile threshold files are
generated by the extraction notebook in the Paper 1 repository. Copy
`data/arrays/` and `data/percentiles/` from that repository.

### ECMWF ensemble data
Raw GRIB ensemble files (one per member, per variable, per run date) are stored
on JASMIN scratch space. The `03_ecmwf_ensemble_processing.ipynb` notebook
reads these directly; set `ECMWF_DATA_DIR` to the appropriate path.

The percentile threshold cubes in `data/percentile_cubes/` are the HadUK-Grid
thresholds regridded to the ECMWF grid using bilinear interpolation.

### ECMWF WP ensemble CSV files
CSV files of the format `Ensemble_Matrix15_{DDMMYYYY}_00_ecmwf_30regimes.csv`
contain a 15 × 51 matrix of weather-pattern assignments (lead days × members).
These are produced by the Met Office Decider system. Place them in
`data/forecasts/{EventName}/`.

### WP subset dictionaries
Per-WP pickle files (`wp_{N}_subsetA_data.pkl`) contain metadata for the
subset of incident days associated with each WP. These are produced by
the analysis in the Paper 2 repository.

### Weather-pattern classification
`preliminary_classifications_2024_30regimes.csv` — daily historical WP
classifications used to compute climatological WP probabilities.

### Observed event data
Binary CD/CW classification pickles (`obs_data_{YYYYMMDD}_clean.pkl`) are
produced by `02_sequence_footprints_plotting.ipynb` in the Paper 2 repository.
Copy them to `output/obs/`.

## Notes

- **WP encoding in footprint arrays:** 0 = neither threshold met; 1 = cold only;
  2 = wet only; 3 = cold-wet (CW); 4 = cold-dry (CD).
- **Modal footprint:** the mode is computed across all days in each WP's subset
  independently at each grid cell, giving the most frequently observed
  classification for that pattern.
- **HRPs highlighted:** WPs 19, 27, and 28 are the high-risk patterns most
  associated with compound cold events. Change `HRP_NUMBERS` in notebook 1 and
  `HRP_LIST` in notebook 2 if your analysis identifies a different subset.
- **Lead-time timestep indices** (`TIMESTEP_INDEX` in notebook 4) map each
  nominal lead time to a specific index in the forecast cube's time dimension.
  These depend on the accumulation period and time-step used in notebook 3 and
  should be verified when changing the forecast configuration.
- **`pattern_climatologies` module:** the `combined_pattern_climatology()`
  function returns the sum of monthly climatological probabilities for a given
  list of patterns, providing a baseline for assessing forecast skill.
